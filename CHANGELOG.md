# Changelog

## [0.0.2] — 2026-04-27

### Added
- Распознавание треков по голосовому сообщению через Shazam (shazamio)
- Автоматический поиск распознанного трека на YouTube (первый результат)
- Лимит длительности голосового — до 30 секунд
- Multi-stage Dockerfile с поддержкой M1/M2/M3 (linux/arm64) и серверной linux/amd64

### Changed
- Логика скачивания и отправки аудио вынесена в общий helper `download_and_send_audio` (используется URL-, поиск- и voice-flow)
- Семафор миграция на `async with` — устранён потенциальный double-release

### Fixed
- HTML-инъекция в voice handler: `title`/`artist` от Shazam теперь экранируются
- `voice.duration` None больше не обходит лимит длительности
- Защита от пустых (0 байт) voice-файлов перед вызовом Shazam
- `videos[0]['id']` None-check в voice flow

## [0.3.0] — 2026-04-04

### Added
- Поиск видео через YouTube Data API v3 с fallback на yt-dlp
- Пагинация результатов поиска (30 видео, по 10 на страницу)
- Рассылка сообщений `/broadcast` через FSM диалог (текст, фото, видео)
- Раздельное меню команд для пользователей и админов

## [0.2.0] — 2026-04-02

### Added
- Статистика бота `/stats` (только для админов)
- Уведомления админам о новых пользователях
- Система сбора статистики с SQLite
- Семафор на 15 одновременных загрузок
- Очистка YouTube URL от параметров плейлиста

### Fixed
- Проверка доступа к `/stats`
- Pool параметры для SQLite
- Volume для сохранения БД в Docker

## [0.1.0] — 2026-03-30

### Added
- Базовый функционал: скачивание аудио из YouTube
- Проверка длительности (макс. 30 мин)
- Проверка размера файла (макс. 50 MB)
- Автоматическая очистка файлов после отправки
- Docker и docker-compose конфигурация
